TypeScript란

Microsoft에서 개발, 유지하는 엄격한 문법의 자바스크립트의 슈퍼셋인 오픈소스 프로그래밍 언어이다.
TypeScript로 작성된 코드는 TSC로 컴파일 후 JavaScript로 변환되어 동작되는 방식이다. 요즘 JavaScript를 사용하는 프로젝트에서는 TypeScript의 사용은 필수적이라고 할 수 있다.

1. TypeScript를 활용하는 프론트엔드 프레임워크

1-1. React.js
리액트와 타입스크립트의 호환성은 조흔 편이다. React.js는 가장 많이 사용되는 프론트 프레임워크이며 최근 들어 타입스크립트와의 조합을 선택하는 개발자들이 많아졌다.
리액트 공식홈페이지에서도 타입스크립트를 사용하기 위한 가이드를 제공한다.

1-2. Vue.js
Vue2에서는 타입스크립트를 사용할 수는 있지만 라이브러리의 도움을 받아야하거나 구현 자체가 안되는 문제점이 있었지만 Vue3부터는 타입스크립트를 지원한다. 

1-3. Angular.js
Angular 버전 2부터는 타입스크립트 기반으로 만들어졌고 타입스크립트 사용을 권장한다.

2. TypeScript 특징

- 컴파일 시 정적 타입 체크 ( 코드 작성 단계에서 타입을 체크해 오류 확인 가능 )
- 미리 타입을 체크해 실행 속도가 매우 빠름
- 가독성 향상 (String, Number, Boolean, Array, Any, Void ...등)
- 객체지향 프로그래밍 강화
- Prototype을 사용하지 않아도 된다. Java 클래스처럼 상속, 접근제한자, 캡슐화, 메소드 사용 가능
- 타입스크립트는 파일별로, 한 파일 안에 일부분에도 적용이 가능

높은 수준의 코드 탐색과 디버깅으로 실제로 타입스크립트를 적용한 이후 자바스크립 버그의 15%가 사전에 타입스크립트로 감지할 수 있다는 연구 결과가 있다.
